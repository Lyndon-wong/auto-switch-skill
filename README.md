## 💡 项目简介

在 OpenClaw 多智能体系统中，每个 Agent 绑定固定的 LLM 模型。然而实际场景中，任务的复杂度差异巨大——从简单的问答闲聊到复杂的架构设计和逻辑推理。固定绑定单一模型会导致两个核心问题：

| 问题 | 描述 |
|:-----|:-----|
| **能力不足** | 轻量模型面对复杂推理任务时质量下降 |
| **资源浪费** | 高性能模型处理简单任务时造成不必要的 Token 消耗 |

**Auto-Switch-Skill** 通过「任务×模型适配度矩阵」驱动的智能路由系统，让 Agent 自动选择最合适的模型，在**保证任务质量的同时优化成本**。

## ✨ 核心特性

- 🧠 **智能任务评估** — 模型自评估（Self-Evaluation）识别 11 种任务类型，无需额外 API 调用
- 🔄 **阻尼式切换决策** — 切换势能累积 + 上下文惯性 + 非对称阈值，防止频繁切换
- 🗺️ **适配度矩阵路由** — 基于 Benchmark 数据的任务×模型权重矩阵，安装时自动生成
- 💾 **三层记忆栈** — 战略记忆(L1) + 摘要链(L2) + 工作区(L3)，切换时零关键信息丢失
- 📊 **增量摘要 DAG** — 树状摘要结构 + 重要度衰减，历史信息永不线性衰减
- ⚡ **`/ms` 极简命令** — 6 大分类 18+ 命令，覆盖切换、路由、调试、记忆管理全场景
- 🛡️ **多重安全兜底** — 冷却期、切换上限、校验回退、`[PIN]` 永不过期保护

### 切换决策流程

```
用户消息 → [冷却期检查] → [模型自评估] → [适配度矩阵查询]
                                              │
                                    权重差值 > 阈值？
                                     │            │
                                    否:衰减      是:累积势能
                                                  │
                                          [上下文惯性检查]
                                                  │
                                          势能 >= 最终阈值？
                                           │            │
                                          否:等待      是:✅ 执行切换
```

### 三层记忆栈

```
┌───────────────────────────────────────────────────────┐
│  L1 战略层  │ ≤2000 tokens │ 跨会话持久 │ 极少更新     │
│  核心决策、架构约定、用户偏好、[PIN] 永不过期          │
├───────────────────────────────────────────────────────┤
│  L2 摘要链  │ ≤4000 tokens │ 按切换更新 │ 渐进合并     │
│  增量摘要 DAG，每次切换生成，历史逐步合并压缩          │
├───────────────────────────────────────────────────────┤
│  L3 工作区  │ 动态分配     │ 当前会话   │ 切换时清理   │
│  最近 N 轮完整对话、当前 plan/todo、代码片段           │
└───────────────────────────────────────────────────────┘
```

## 🚀 快速开始

### 环境要求

- Python 3.11+
- [OpenClaw](https://github.com/anthropics/openclaw) 多智能体框架

### 安装

```bash
# 克隆仓库
git clone https://github.com/Lyndon-wong/auto-switch-skill.git

# 进入项目目录
cd auto-switch-skill
```

### 集成到 OpenClaw

安装时系统会自动执行以下流程：

1. 读取 `openclaw.json` 中已注册的可用模型列表
2. 与内置画像库 `model_profiles.yaml` 模糊匹配
3. 生成用户专属的 `routing_matrix.yaml`
4. 输出摘要表格供用户确认

## 📋 命令参考

通过 `/ms`（或 `/model-switch`）命令管理所有功能：

### 模式与切换

| 命令 | 说明 |
|:-----|:-----|
| `/ms auto` | 恢复自动切换模式 |
| `/ms manual` | 切换到手动模式（禁用自动切换） |
| `/ms <模型名>` | 切换到指定模型，不改变当前模式 |

### 路由管理

| 命令 | 说明 |
|:-----|:-----|
| `/ms router` | 查看当前任务模型路由表 |
| `/ms set <模型> <任务缩写> <适配度>` | 修改某模型某任务的权重（0-100） |
| `/ms reset` | 从画像库重新生成路由矩阵 |

### 调试与监控

| 命令 | 说明 |
|:-----|:-----|
| `/ms debug [on\|off]` | 开启/关闭调试模式（回答末尾追加运行时脚注） |
| `/ms history` | 查看本会话的模型切换历史 |
| `/ms stats` | 查看切换统计（次数、成本、各模型使用占比） |
| `/ms status` | 查看当前运行状态 |

### 记忆管理

| 命令 | 说明 |
|:-----|:-----|
| `/ms memory` | 查看三层记忆栈使用状态 |
| `/ms pin <内容>` | 手动将内容 PIN 到 L1 战略记忆（永不过期） |

### 配置管理

| 命令 | 说明 |
|:-----|:-----|
| `/ms cooldown <轮数>` | 设置切换后冷却轮数 |
| `/ms threshold <upgrade\|downgrade> <值>` | 调整升级/降级触发阈值 |
| `/ms export` | 导出当前配置为 YAML |
| `/ms import <文件路径>` | 从 YAML 导入配置 |

> 💡 任务缩写对照：`CHAT` `QA` `SUM` `TRA` `CS` `CC` `REA` `ANA` `CRE` `PLN` `MS`

## 🔬 技术细节

### 为什么不能跨模型复用 KV Cache？

不同模型间无法直接复用 KV Cache，原因包括：
- **架构差异**：层数、隐藏层维度、注意力头数不兼容
- **Tokenizer 不同**：Token 向量表示完全不同
- **语义不一致**：不同模型对同一段文字的注意力权重分配不同

因此，Auto-Switch-Skill 采用**文件系统作为持久化记忆**的策略（三层记忆栈 + MEMORY.md），而非依赖内部缓存。

### 阻尼系统设计理念

升级敏感，降级保守：

| 切换方向 | 阈值 | 每轮增量 | 约需轮数 | 设计原因 |
|:---------|:-----|:---------|:---------|:---------|
| **升级** | 50 | +40 | ~2 轮 | 能力不足直接影响输出质量 |
| **降级** | 80 | +20 | ~4-5 轮 | 大模型处理小任务不出错，只浪费成本 |
